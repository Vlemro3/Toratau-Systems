# Инструкция по публичному деплою Landing Page

## Проблема с ngrok
Ngrok блокирует ваш IP адрес (136.169.149.249). Используем альтернативные методы.

## Вариант 1: Vercel (Рекомендуется - самый простой)

### Шаг 1: Подготовка
1. Убедитесь, что папка `frontend/dist` содержит собранные файлы
2. Файл `vercel.json` уже создан в проекте

### Шаг 2: Деплой через веб-интерфейс

1. **Зарегистрируйтесь/войдите** на https://vercel.com
2. **Создайте новый проект:**
   - Нажмите "Add New" → "Project"
   - Импортируйте репозиторий (если есть на GitHub) ИЛИ
   - Используйте "Deploy" → "Upload" (загрузите папку `dist`)

3. **Если загружаете папку dist:**
   - Перейдите на https://vercel.com/new
   - Выберите "Deploy" → "Upload"
   - Перетащите папку `frontend/dist`
   - Vercel автоматически определит настройки

4. **Настройки проекта:**
   - Framework Preset: Other
   - Root Directory: (оставьте пустым или укажите `dist`)
   - Build Command: (не нужен, файлы уже собраны)
   - Output Directory: (оставьте пустым)

5. **Деплой:**
   - Нажмите "Deploy"
   - Через 1-2 минуты получите публичную ссылку вида: `https://your-project.vercel.app`

### Шаг 3: Получите ссылку
После деплоя Vercel даст вам постоянную публичную ссылку, доступную из любой точки мира!

## Вариант 2: Netlify Drop (Еще проще)

1. Перейдите на https://app.netlify.com/drop
2. Перетащите папку `frontend/dist` в браузер
3. Получите мгновенную ссылку вида: `https://random-name-12345.netlify.app`

**Преимущества:**
- Не требует регистрации для быстрого теста
- Мгновенный деплой
- HTTPS автоматически

## Вариант 3: GitHub Pages (Если есть репозиторий)

1. Закоммитьте и запушьте код в GitHub
2. В настройках репозитория включите GitHub Pages
3. Выберите папку `dist` как source
4. Получите ссылку: `https://username.github.io/repo-name`
